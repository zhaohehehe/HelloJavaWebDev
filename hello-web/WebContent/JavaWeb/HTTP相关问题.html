1. telnet服务
    1. 首先安装telnet服务
    2. 在浏览器地址栏输入网址访问网页时，采用的都是GET方法获取资源
       (1)命令行输入 telnet ip或者域名  端口号
 (2)输入以下命令：
	HEAD /index.html HTTP/1.1
	Host:www.sina.com.cn 
	
	GET / HTTP/1.1 //默认首页
	Host:localhost
	Connection:Keep-Alive //持续连接（http 1.1默认连接方式）
	Connection:close //断开连接
  (3)输入完成后：敲2次回车
  2. get/post的本质区别：底层实现不同
    1. GET:
          (1)请求URL包含请求参数: GET /HelloWebWorld/LoginServlet?username=hello&password=world HTTP/1.1
    2. POST:
          (1)请求URL: POST /HelloWebWorld/LoginServlet HTTP/1.1
          (2)请求体包含请求参数: Connection:Keep-Alive(CRLF)(CRLF)    username=hello&password=world
    3. 结论：因为url长度有限，所以get资源大小有限，上传文件要使用post
    4. 浏览器永远不能和Servlet直接打交道，一定要通过服务器（Tomcat）

  3. tomcat服务器实现了HttpServletRequest req, HttpServletResponse resp（接口），所以我们可以按照一定的规则进行Web开发

  4. 绝对路径和相对路径
    1. 绝对路径：action="/HelloWebWorld/LoginServlet"：HelloWebWorld是死的，一旦在tomcat配置文件中有改动，这里也必须改变。
    2. 相对路径（推荐使用）：action="LoginServlet"：相对于当前项目，即使tomcat的path变化不影响。
